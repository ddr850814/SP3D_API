### 1. 交付物树构建模板

#### 1.1 获取图纸根节点并填充 TreeView

核心入口是 `FillTreeView()`，从 `RootDeliverable` 出发构建整棵交付物树：

```csharp
// 连接站点并获取图纸根节点
oSite = MiddleServiceProvider.SiteMgr.ConnectSite();

// 获取图纸根交付物（整个图纸体系的入口）
IDeliverableParent bookParent = 
    MiddleServiceProvider.SiteMgr.ActiveSite.ActivePlant.PlantModel.RootDeliverable;

TreeNode parentNode = new TreeNode(PlantName);
treeViewDeliverableRoot.Nodes.Add(parentNode);

// 获取子组件集合（Folder / ImportedFolder / SearchFolder 等容器节点）
IReadOnlyCollection<IDeliverableChild> childrenBOs = bookParent.DeliverableComponents;

foreach (BusinessObject oChild in childrenBOs)
{
    TreeNode treeComponent = new TreeNode(oChild.ToString());
    parentNode.Nodes.Add(treeComponent);
    AddFirstChild(treeComponent);  // 递归加载第一个子节点用于预览展开
}
```

#### 1.2 双层子节点集合

每个 `IDeliverableParent` 同时拥有两类子集合：

| 集合属性 | 类型 | 含义 |
|----------|------|------|
| `DeliverableComponents` | `IReadOnlyCollection<IDeliverableChild>` | 子组件（文件夹等容器） |
| `Deliverables` | `IReadOnlyCollection<IDeliverableChild>` | 子交付物（图纸、报告等叶子） |

```csharp

IReadOnlyCollection<IDeliverableChild> oComponentCol = oDocParent.DeliverableComponents; // 子文件夹
```

#### 1.3 GetObjectByPath — 按路径定位业务对象（静态方法）

这是 DemoApp 最关键的辅助方法，通过 TreeView 节点的 `FullPath` 定位到对应的业务对象。内部根据 `isComponent` 参数区分查找组件还是交付物：

```csharp
public static BusinessObject GetObjectByPath(String path, bool isComponent = true)
{
    String newPath = path;
    String plantName = MiddleServiceProvider.SiteMgr.ActiveSite.ActivePlant.Name;

    // 去除路径中的工厂名前缀
    if (path.StartsWith(plantName))
        newPath = path.Remove(0, plantName.Count());

    if (string.IsNullOrEmpty(newPath))
        return (BusinessObject)MiddleServiceProvider.SiteMgr.ActiveSite.ActivePlant
                                    .PlantModel.RootDeliverable;
    else if (isComponent == true)
    {
        IDeliverableParent component = null;
        // 尝试按路径获取组件
        DeliverableRoot.TryGetDeliverableComponent(newPath, out component);
        return (BusinessObject)component;
    }
    else
    {
        IDeliverableChild deliverable = null;
        // 尝试按路径获取交付物（图纸/报告）
        DeliverableRoot.TryGetDeliverable(newPath, out deliverable);
        return (BusinessObject)deliverable;
    }
}
```

| 方法 | 用途 |
|------|------|
| `DeliverableRoot.TryGetDeliverableComponent(path, out component)` | 按路径查找容器组件 |
| `DeliverableRoot.TryGetDeliverable(path, out deliverable)` | 按路径查找交付物（可能抛 `DuplicateDeliverableFoundException`） |

> **注意**: `GetDeliverableObjectByPath` 封装方法先尝试组件查找，找不到再尝试交付物查找，确保两种节点都能定位。

#### 2.2 CreatePropertyValue — 按类型创建属性值（完整 switch）

`CreatePropertyValue` 方法根据 `SP3DPropType` 枚举分发到对应类型的 `PropertyValue` 构造器，是属性写入的核心工厂方法：

```csharp
private PropertyValue CreatePropertyValue(String interfaceName, String propertyName, 
    String value, SP3DPropType oPropType)
{
    switch (oPropType)
    {
        case SP3DPropType.PTUndefined:
            return null;

        case SP3DPropType.PTInteger:
            int valInt = Convert.ToInt16(value);
            return new PropertyValueInt(interfaceName, propertyName, valInt);

        case SP3DPropType.PTString:
            return new PropertyValueString(interfaceName, propertyName, value);

        case SP3DPropType.PTBool:
            Boolean valBoolean = Convert.ToBoolean(value);
            return new PropertyValueBoolean(interfaceName, propertyName, valBoolean);

        case SP3DPropType.PTDate:
            DateTime valDate = Convert.ToDateTime(value);
            return new PropertyValueDateTime(interfaceName, propertyName, valDate);

        case SP3DPropType.PTDouble:
            Double valDouble = Convert.ToDouble(value);
            return new PropertyValueDouble(interfaceName, propertyName, valDouble);

        case SP3DPropType.PTCodelist:
            int valCodelist = Convert.ToInt16(value);
            return new PropertyValueCodelist(interfaceName, propertyName, valCodelist);

        case SP3DPropType.PTShort:
            short valShort = Convert.ToInt16(value);
            return new PropertyValueShort(interfaceName, propertyName, valShort);

        case SP3DPropType.PTFloat:
            float valFloat = Convert.ToSingle(value);
            return new PropertyValueFloat(interfaceName, propertyName, valFloat);
    }
    return null;
}
```

| SP3DPropType | PropertyValue 类型 | 构造签名 |
|--------------|-------------------|----------|
| `PTInteger` | `PropertyValueInt` | `new PropertyValueInt(interfaceName, propertyName, int)` |
| `PTString` | `PropertyValueString` | `new PropertyValueString(interfaceName, propertyName, string)` |
| `PTBool` | `PropertyValueBoolean` | `new PropertyValueBoolean(interfaceName, propertyName, bool)` |
| `PTDate` | `PropertyValueDateTime` | `new PropertyValueDateTime(interfaceName, propertyName, DateTime)` |
| `PTDouble` | `PropertyValueDouble` | `new PropertyValueDouble(interfaceName, propertyName, double)` |
| `PTCodelist` | `PropertyValueCodelist` | `new PropertyValueCodelist(interfaceName, propertyName, int)` |
| `PTShort` | `PropertyValueShort` | `new PropertyValueShort(interfaceName, propertyName, short)` |
| `PTFloat` | `PropertyValueFloat` | `new PropertyValueFloat(interfaceName, propertyName, float)` |

#### 2.3 两种属性写入模式

| 模式 | 方法 | 说明 |
|------|------|------|
| 单属性写入 | `propInfo.SetValue(oEntity, value)` | 通过反射逐个设置，`Behavior` 列只读 |
| 多属性批量写入 | `oEntity.SetProperties(listModifiedProperties)` | 传入 `List<DeliverableProperty>`，`Behavior` 可编辑 |


### 3. 版本/发布记录

#### 3.1 RevisionRecord — 版本记录

版本记录（RevisionRecord）仅存在于 `Deliverable`（图纸）对象上，通过 `SupportsInterface("IJDDwgSheet2")` 判断节点是否为图纸：

```csharp
private RevisionRecord GetRevisionRecord()
{
    Deliverable sheet = null;
    RevisionRecord record = null;

    oEntity = (DeliverableBase)GetObjectByPath(NodePath, false);

    // 判断是否为图纸对象
    if (oEntity.SupportsInterface("IJDDwgSheet2") == true)
        sheet = oEntity as Deliverable;
    else
        MessageBox.Show("Record is associated with drawing sheet only not with component");

    if (sheet != null)
        record = sheet.GetLastRevisionRecord();

    return record;
}
```

### 4. SaveAs 导出工作流

#### 4.1 三种 SaveAs 重载

`DeliverableBase.SaveAs` 提供三个重载，根据传入参数自动选择：

```csharp
ReadOnlyCollection<System.IO.FileInfo> files = null;

// 重载1：指定导出格式（AutoCAD/MicroStation）
if (form.SaveAsExtension != 0)
{
    files = deliverableBase.SaveAs(
        form.FolderPath,          // 目标文件夹路径
        form.UseModelHierarchy,   // 是否使用模型层级
        form.SaveAsExtension);    // TargetFileType 枚举
}
// 重载2：默认格式，无输出选项
else if (outputOptions.Count < 1)
{
    files = deliverableBase.SaveAs(form.FolderPath, form.UseModelHierarchy);
}
// 重载3：指定输出文档类型组合（OutputType 位标志）
else
{
    files = deliverableBase.SaveAs(
        form.FolderPath,
```

### 6. 搜索文件夹 (SearchChildForm)

`SearchChildForm` 为 `SearchFolder` 配置过滤器和搜索根节点，支持三种过滤器来源。

#### 6.1 三种过滤器类型

```csharp
private FilterBase CreateDrawingSheetsObjectTypeFilter(string name, 
    bool isMyFilter = false, bool isParamteric = false)
{
    MiddleFiltersFactory factory = new MiddleFiltersFactory();

    // 选择过滤器根文件夹类型
    MFRootFolderTypes type = MFRootFolderTypes.MFPlantFiltersRoot;      // 工厂过滤器
    if (isMyFilter)
        type = MFRootFolderTypes.MFMyFiltersRoot;                       // 我的过滤器

    IJFilterFolder comFilterFolder = factory.GetRootFolder(type, true, null);
    FilterFolder filterFolder = (FilterFolder)COMConverters.ConvertCOMBOToBO(comFilterFolder);

    // 检查是否已存在同名过滤器
    FilterBase existingFilter = GetExistingFilter(filterFolder, name);
    if (existingFilter == null)
    {
        Filter drawingObjectTypeFilter = new Filter(name, filterFolder);

        if (isParamteric)
            drawingObjectTypeFilter.Definition.AddParametricObjectType();  // 参数化对象
        else
            drawingObjectTypeFilter.Definition.AddObjectType(@"DrawingAndReport\DrawingSheet");

        return drawingObjectTypeFilter;
    }
    return existingFilter;
}
```

| 过滤器类型 | MFRootFolderTypes | Definition 方法 | 说明 |
|-----------|-------------------|----------------|------|
| 工厂过滤器 | `MFPlantFiltersRoot` | `AddObjectType(@"DrawingAndReport\DrawingSheet")` | 图纸表对象 |
| 我的过滤器 | `MFMyFiltersRoot` | `AddObjectType(...)` | 同上，存储位置不同 |
| 参数化过滤器 | `MFPlantFiltersRoot` | `AddParametricObjectType()` | 参数化对象类型 |

## 二、Reports — 自定义查询解释器 (CustomQueryInterpreter)

> **Path**: `Reports\SOM\Examples\CustomQueryInterpreter\CustomLabelQI\`
> **Language**: C# | **Namespace**: `Ingr.SP3D.Reports.Content.Examples`
> **功能**: 演示如何通过继承 `QueryInterpreter` 基类开发自定义报表/标签查询解释器，在委托查询结果基础上追加自定义字段。

### 1. 查询解释器开发模板

#### 1.1 基类继承与 Execute 方法重写

所有自定义查询解释器都继承自 `QueryInterpreter`，核心是重写 `Execute` 方法：

```csharp
public class MyQueryInterpreter : QueryInterpreter
{
    private DataTable m_DataTable;

    public MyQueryInterpreter() : base() { }

    public override DataTable Execute(string action, string argument)
    {
        try
        {
            // 步骤1: 定义输出列
            List<Column> FieldColumnList = new List<Column>();
            FieldColumnList.Add(new Column("OID", typeof(System.String)));
            FieldColumnList.Add(new Column("name", typeof(System.String)));

            // 步骤2: 初始化 DataTable（合并自定义列 + 委托查询返回的列）
            m_DataTable = InitializeDataTable(FieldColumnList);

            // 步骤3: 设计时模式 — 仅返回列名供 Query Builder 显示
            if (EvaluateOnly)
                return m_DataTable;

            // 步骤4: 遍历输入对象，填充数据
            foreach (BusinessObject oBO in InputObjects)
            {
                DataRow currentRow = m_DataTable.NewRow();
                currentRow.SetField("OID", oBO.ObjectID.ToString());
                currentRow.SetField("name", oBO.ToString());

                // 步骤5: 执行委托查询并合并结果
                DataTable DelegatedDataTable = ExecuteDelegatedQuery(oBO);
                if (DelegatedDataTable != null && DelegatedDataTable.Rows.Count >= 1)
                {
                    DataRow row = DelegatedDataTable.Rows[0];
                    foreach (DataColumn col in row.Table.Columns)
                    {
                        if (col.ColumnName.ToLower() != "oid")
                            currentRow.SetField(col.ColumnName, row[col.ColumnName]);
                    }
                }
                m_DataTable.Rows.Add(currentRow);
            }
            return m_DataTable;
        }
        catch (Exception e)
        {
            MiddleServiceProvider.ErrorLogger.Log(0, typeof(MyQueryInterpreter).FullName,
                e.ToString(), string.Empty,
                (new StackTrace()).GetFrame(1).GetMethod().Name,
                string.Empty, string.Empty, -1);
            throw;
        }
    }
}
```

#### 1.2 Execute 方法标准流程

```
Execute(action, argument)
  │
  ├── 定义自定义列 → InitializeDataTable(columnList)
  │
  ├── if (EvaluateOnly) return m_DataTable;  ← 设计时仅返回列结构
  │
  ├── foreach (oBO in InputObjects)
  │     ├── 创建 DataRow
  │     ├── 填充自定义列值
  │     ├── ExecuteDelegatedQuery(oBO) → 获取委托查询结果

  │
  └── return m_DataTable;
```

#### 2.1 ExampleQueryInterpreter — 标签属性查询

最基础的查询解释器示例，演示如何为每个输入对象提取属性并合并委托查询结果：

```csharp
public override DataTable Execute(string action, string argument)
{
    // 定义输出列
    List<Column> FieldColumnList = new List<Column>();
    FieldColumnList.Add(new Column("OID", typeof(System.String)));
    FieldColumnList.Add(new Column("name", typeof(System.String)));
    FieldColumnList.Add(new Column("Dry Weight", typeof(System.Double)));
    FieldColumnList.Add(new Column("Date Created", typeof(System.DateTime)));
    FieldColumnList.Add(new Column("Approval State", typeof(System.String)));
    FieldColumnList.Add(new Column("DotNetLabel", typeof(System.String)));

    m_DataTable = InitializeDataTable(FieldColumnList);

    if (EvaluateOnly) return m_DataTable;

    foreach (BusinessObject oBO in InputObjects)
    {
        double dryWeightVal = 0.0;
        // 检查对象是否支持 IJWeightCG 接口
        if (oBO.SupportsInterface("IJWeightCG"))
        {
            PropertyValueDouble dblWeight = (PropertyValueDouble)
                oBO.GetPropertyValue("IJWeightCG", "DryWeight");
            if (null != dblWeight.PropValue)
                dryWeightVal = (double)dblWeight.PropValue;
        }

        DataRow currentRow = m_DataTable.NewRow();
        currentRow.SetField("OID", oBO.ObjectID.ToString());
        currentRow.SetField("name", oBO.ToString());
        currentRow.SetField("Dry Weight", dryWeightVal);
        currentRow.SetField("Date Created", oBO.DateCreated.Date);  // 去除时间部分
        currentRow.SetField("Approval State", oBO.ApprovalStatus.ToString());
        currentRow.SetField("DotNetLabel", "[Equipment Properties]");

        // 合并委托查询结果
        DataTable DelegatedDataTable = ExecuteDelegatedQuery(oBO);
        // ... 合并逻辑 ...
        m_DataTable.Rows.Add(currentRow);
    }
    return m_DataTable;
}
```

### 3. 关键 API

| API | 说明 |
|-----|------|
| `InputObjects` | 输入对象集合（`BusinessObject` 列表），运行时由框架传入 |
| `ExecuteDelegatedQuery()` | 执行委托查询（无参数版本），返回基础 `DataTable` |
| `ExecuteDelegatedQuery(businessObject)` | 对指定业务对象执行委托查询 |
| `SetQuery(sql)` | 设置委托查询的 SQL 语句 |
| `SetQuery(sql, dbProviderType)` | 为指定数据库类型（MSSQL/Oracle）设置 SQL |
| `GetParameterValueByName("OID")` | 按名称获取查询参数值 |
| `InitializeDataTable(columnList)` | 用自定义列 + 委托查询列初始化 DataTable |
| `EvaluateOnly` | 设计时模式标志（true 时仅返回列结构，不执行数据查询） |
| `AddRow(dictionary)` | 添加一行数据到 DataTable（键值对方式） |
| `WrapSP3DBO(moniker)` | 将数据库标识符包装为业务对象 |
| `GetBOMonikerFromDbIdentifier(oid)` | 从 OID 获取业务对象 Moniker |
| `ErrorLogger.Log(...)` | 错误日志记录 |
| `SupportsInterface("IJWeightCG")` | 检查对象是否支持指定接口 |
| `GetPropertyValue(interfaceName, propertyName)` | 获取属性值 |

## 三、开发要点总结

### Drawings 开发要点

1. **连接管理**: 使用 `MiddleServiceProvider.SiteMgr.ConnectSite()` 连接，窗体关闭时 `Abort()` + `Cleanup()`
2. **事务模式**: 每个操作后必须 `Commit(description)` 或 `Abort()`，不能遗留未提交事务
3. **路径定位**: `GetObjectByPath` 是核心辅助方法，注意组件和交付物的查找路径不同
4. **属性写入**: 批量写入用 `SetProperties`，单属性写入用反射 `SetValue`；Behavior 从 Inherit 改为 Override 时需手动切换
5. **SaveAs 后必须 Abort**: 导出操作产生的事务不需要持久化

### Reports 开发要点

1. **EvaluateOnly 检查**: 必须在设计时（`EvaluateOnly == true`）提前返回，否则 Query Builder 无法显示列
2. **委托查询合并**: 自定义列和委托查询列通过 `InitializeDataTable` 自动合并，注意排除 OID 列避免重复
3. **多数据库支持**: 使用 `SetQuery(sql, dbProviderType)` 同时为 MSSQL 和 Oracle 设置 SQL
4. **对象绑定**: `WrapSP3DBO` + `GetBOMonikerFromDbIdentifier` 是从 OID 获取业务对象的标准方式
5. **错误日志**: 所有异常必须通过 `ErrorLogger.Log` 记录后重新抛出
